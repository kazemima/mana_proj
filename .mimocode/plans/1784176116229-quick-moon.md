# پیاده‌سازی ویژگی‌های مدیریت کاربران

## خلاصه
۴ ویژگی جدید به پنل مدیریت اضافه می‌شود:
1. فرم مودال برای افزودن/ویرایش کاربر
2. سطوح دسترسی (admin, editor, viewer)
3. فیلد تکرار رمز عبور
4. بازیابی رمز عبور از طریق ایمیل

---

## ۱. فرم مودال افزودن/ویرایش کاربر

**فایل‌ها:** `admin/users.php`, `assets/css/admin.css`, `assets/js/admin.js`

### تغییرات:
- **admin/users.php**: فرم موجود حذف شده و به صورت HTML مودال بازنویسی می‌شود
- فرم در انتهای صفحه (قبل از footer) قرار می‌گیرد
- جدول کاربران دکمه‌های "افزودن کاربر جدید" و "ویرایش" دارد که مودال را باز می‌کنند
- اطلاعات کاربر برای ویرایش از طریق `data-*` attribute روی دکمه‌ها ارسال می‌شود
- فیلدهای مودال: نام کاربری، نام کامل، ایمیل، رمز عبور، تکرار رمز، نقش

### اسکریپت (admin.js):
- تابع `openUserModal(userData?)` — فرم را پر/خالی و نمایش می‌دهد
- تابع `closeUserModal()` — بستن مودال
- اعتبارسنجی client-side: تطابق رمز عبور

---

## ۲. سطوح دسترسی

**فایل‌ها:** `admin/users.php`, `assets/js/admin.js`

### نقش‌ها:
| نقش | مقدار | توضیح |
|-----|-------|-------|
| مدیر | admin | دسترسی کامل |
| ویرایشگر | editor | ویرایش محتوا، بدون حذف کاربر |
| مشاهده‌گر | viewer | فقط مشاهده، بدون ویرایش |

### تغییرات:
- **users.php**: select نقش سه گزینه دارد (admin, editor, viewer)
- **users.php**: جدول نقش‌ها را با badge نمایش می‌دهد (admin=green, editor=blue, viewer=gray)
- **users.php**: حذف کاربر فقط برای admin امکان‌پذیر است
- **functions.php**: تابع `hasPermission($requiredRole)` اضافه شود

---

## ۳. فیلد تکرار رمز عبور

**فایل‌ها:** `admin/users.php`, `assets/js/admin.js`

### تغییرات:
- در مودال فرم کاربر، فیلد "تکرار رمز عبور" اضافه شود
- اعتبارسنجی client-side: اگر رمز و تکرار یکی نبود، فرم ارسال نشود
- اعتبارسercity server-side: بررسی قبل از ذخیره

---

## ۴. بازیابی رمز عبور از طریق ایمیل

**فایل‌ها:** `admin/login.php`, `admin/forgot-password.php` (جدید), `admin/reset-password.php` (جدید), `includes/database.php`

### مراحل کاربر:
1. کاربر روی "فراموشی رمز" در صفحه لاگین کلیک می‌کند
2. ایمیل خود را وارد می‌کند
3. سیستم یک توکن یکبار مصرف تولید کرده و لینک بازیابی به ایمیل ارسال می‌کند
4. کاربر روی لینک کلیک کرده و رمز جدید تعیین می‌کند

### تغییرات دیتابیس:
- جدول `password_resets` اضافه شود:
  ```sql
  CREATE TABLE IF NOT EXISTS password_resets (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      email TEXT NOT NULL,
      token TEXT NOT NULL,
      created_at DATETIME DEFAULT CURRENT_TIMESTAMP
  );
  ```

### فایل‌های جدید:
- **admin/forgot-password.php**: فرم درج ایمیل + ارسال لینک بازیابی
- **admin/reset-password.php**: فرم رمز جدید + اعتبارسنجی توکن

### تغییرات admin/login.php:
- لینک "رمز عبور خود را فراموش کرده‌اید؟" اضافه شود

### ارسال ایمیل:
- از تابع PHP `mail()` استفاده می‌شود (بدون نیاز به کتابخانه اضافی)
- توکن: `bin2hex(random_bytes(32))`
- اعتبار توکن: ۱ ساعت

---

## ترتیب اجرا

1. اضافه کردن جدول `password_resets` به `database.php`
2. اضافه کردن تابع `hasPermission()` به `functions.php`
3. بازنویسی `admin/users.php` با مودال + نقش‌ها + تکرار رمز
4. اضافه کردن کد مودال به `assets/js/admin.js`
5. اضافه کردن استایل مودال به `assets/css/admin.css` (موجود — بررسی و تکمیل)
6. ایجاد `admin/forgot-password.php`
7. ایجاد `admin/reset-password.php`
8. به‌روزرسانی `admin/login.php` با لینک فراموشی رمز

---

## احراز هویت و تست

1. صفحه کاربران را باز کنید — جدول نمایش داده شود
2. دکمه "افزودن کاربر" — مودال باز شود
3. فرم را پر کنید — تکرار رمز مطابقت نداشته باشد — خطا نمایش داده شود
4. کاربر اضافه شود و در جدول نمایش داده شود
5. دکمه ویرایش — مودال با اطلاعات کاربر باز شود
6. نقش viewer را تست کنید — کاربر viewer نتواند حذف کند
7. صفحه لاگین — "فراموشی رمز" — ایمیل ارسال شود
8. لینک بازیابی — رمز جدید — لاگین با رمز جدید
